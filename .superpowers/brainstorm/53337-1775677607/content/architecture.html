Welke architectuur past bij dit project?

Nu is het 1 groot index.html bestand (1.2MB). Dat schaalt niet naar honderden restaurants.

A

Static site + API calls (huidige stijl)

Houd het simpel: vanilla HTML/CSS/JS maar met echte API calls naar Formitable. Restaurants in een JSON bestand of kleine database. Geen framework, geen build step. Pro: simpel, snel, gratis hosting op Vercel. Con: wordt rommelig bij veel features.

B

Next.js (React) + Vercel

Modern framework met server-side rendering, API routes voor Formitable proxy, en component-based UI. Pro: schaalt goed, SEO-friendly, API routes ingebouwd. Con: meer setup, React kennis nodig, meer complexity.

C

Vanilla frontend + Vercel Serverless backend

Houd de frontend simpel (zoals nu) maar voeg Vercel serverless functions toe als backend. API keys veilig server-side, caching van beschikbaarheid, restaurant database als JSON/Supabase. Pro: best of both worlds — simpele frontend, krachtige backend. Con: twee werelden om te beheren.
