Design — Sectie 5: Data Flows

Hoe data door het systeem stroomt — de drie belangrijkste flows

Flow 1: Gebruiker zoekt beschikbaarheid
Gebruiker
kiest datum + personen
→
Next.js frontend
fetch /api/restaurants
→
API route
query Supabase + filters
→
Per restaurant
fetch Formitable slots
→
Response
restaurants + beschikbaarheid
Caching: Formitable beschikbaarheid wordt 5 minuten gecached in Vercel KV. Voorkomt dat we Formitable overbelasten en maakt herhaalde zoekopdrachten instant.
Flow 2: Restaurant import (eenmalig/periodiek)
Admin trigger
/api/import
→
Google Places
"restaurants in Amsterdam"
→
Verwerk resultaten
naam, adres, foto, rating
→
Match Formitable
check widget beschikbaar?
→
Supabase
upsert restaurants
Frequentie: Eerste keer bulk import, daarna wekelijks/maandelijks om nieuwe restaurants te ontdekken. Bestaande data wordt niet overschreven (upsert op google_place_id).
Flow 3: Reservering maken
Gebruiker klikt
tijdslot of restaurant
→
BookingModal
opent op basis van type
→
formitable: embedded widget iframe
link: open externe reserveer-URL
manual: bel/mail opties tonen
Belangrijk: Wij verwerken geen reserveringen zelf. We linken door naar Formitable, de restaurant-website, of tonen contactgegevens. Zero liability.
