Waar slaan we restaurant-data op?

Nu staan 10 restaurants hardcoded in index.html. Bij honderden restaurants heb je een echte databron nodig.

A

JSON bestand in de repo

Een restaurants.json met alle data. Makkelijk te bewerken, geen database nodig. Nieuwe restaurants = PR maken. Pro: zero cost, simpel, versie-beheer via git. Con: handmatig bijwerken, geen admin panel, geen dynamische updates.

B

Supabase (Postgres)

Gratis hosted database. Restaurant tabel met alle info. Later makkelijk uit te breiden met users, reviews, favorieten. Pro: schaalbaar, admin mogelijk, real-time updates. Con: extra dependency, setup nodig.

C

Start JSON, migreer later naar Supabase

Begin met een JSON file zodat je snel live bent. Zodra je een admin panel of user features wilt, migreer naar Supabase. Pro: snel starten, geen vendor lock-in. Con: migratie-moment later.
